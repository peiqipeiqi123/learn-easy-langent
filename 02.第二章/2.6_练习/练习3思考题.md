# 练习3 思考题

> 若开发一个"结构化报告生成工具"（需固定提示格式、输出结构化数据，但无需多轮流程），需组合本章哪些核心组件？说明理由。

---

## 我的想法

以学习规划类报告为例：

- 需要一个 **ChatOpenAI**，接收用户提出的规划需求。
- 用 **PromptTemplate** 在系统提示词中限定大模型角色为"规划师"，排除无关内容。
- 用 **FewShotPromptTemplate** 给模型做示例引导。比如用户问"如何备考？"，示例里会展示：先说数学怎么准备，再说英语怎么学，过程中要注意什么。模型照着示例格式输出。
- 最后用 **PydanticOutputParser** 做结构化输出。如果不需要强校验，也可以换成 StrOutputParser。

---


### 需要组合的组件（4个）

| 组件 | 作用 |
|------|------|
| **ChatOpenAI** | 核心引擎，调用大模型生成报告内容 |
| **PromptTemplate** | 固定角色（规划师）和报告结构，保证每次输入格式一致 |
| **FewShotPromptTemplate** | 通过示例引导模型按照规范格式输出，解决"格式不固定"问题 |
| **PydanticOutputParser** | 将模型输出转为带类型校验的结构化数据，方便程序直接取用 |

### 为什么选 Pydantic 而不是 Str？

StrOutputParser 只返回纯文本，无法直接提取"数学建议"、"英语建议"等字段。Pydantic 可以定义数据模型：

```python
class StudyPlan(BaseModel):
    math_advice: str = Field(description="数学备考建议")
    english_advice: str = Field(description="英语备考建议")
    notes: str = Field(description="注意事项")
```

程序可以直接 `result.math_advice` 取字段，无需手动从大段文字中截取。

### 数据流向

```
PromptTemplate（设定角色、固定格式）
    ↓
FewShotPromptTemplate（放入示例引导）
    ↓
ChatOpenAI（调用大模型生成）
    ↓
PydanticOutputParser（校验 → 结构化输出）
```

总结：四者串联形成"输入控制 → 格式引导 → 模型生成 → 结构校验"的完整链路，实现输入可控制、输出可预期。
